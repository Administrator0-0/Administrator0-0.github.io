java层消息机制

Java层Android消息机制

1.什么是消息？

​ 线程与线程之间跨线程的通讯，而这个通讯内容就是所谓的消息。

2.为什么需要消息机制？

​ 在不同的线程中要实现通讯，比如：由于主线程是不能有耗时操作的（避免ANR），于是耗时操作都在子线程中处理，那么在子线程中耗时操作处理完后需要更新UI，这时就需要消息机制将这个消息送到主线程中处理。

3.组成

  • Message：消息。

  • MessageQueue：消息队列，存储着待处理的消息，其虽叫做队列，但内部采用了单链表的数据结构。

  • Handler：消息处理器，发送消息和处理消息。

  • Looper：轮询器，从MessageQueue（消息队列）中取消息，分发给Handler处理。每个线程有一个Looper对象，主线程自动创建，子线程需要自行创建，由Looper管理消息队列。

4. Message

  • obj：Object类型字段。

  • what：消息标志，可以根据标志进行不同的处理。

    一般使用Message.obtain()或者Handler.obtainMessage()（实际上调用了Message.obtain()）获得一个Message对象，该方法会先从消息池中寻找可用的消息，如果有直接复用，否则new一个新的消息对象。

5. MessageQueue

主要是enqueueMessage()入队方法和next()出队方法。

enqueueMessage(Message msg, long when)将新的消息按照when从小到大的方式插入。

next()使用一个死循环遍历链表，若当前链表内没有消息，就会把mBlocked置为true并阻塞，否则会删去链表第一个任务并返回。

6. Looper
Looper的作用是不断地遍历消息队列，有消息就拿出来交给Handler处理，若没有消息就处于阻塞状态。

Looper在构造的时候便会创建一个消息队列，并将当前线程赋值给成员变量mThread。在新建Handler时一定要调用Looper.prepare()。在该方法中最终新建了Looper并放入了当前线程的ThreadLocal中保存。

Looper通过loop()方法开启消息循环

loop方法中使用了一个死循环来进行消息循环，首先是调用了消息队列的next方法，该方法如果消息队列为空的话会一直阻塞从而导致Looper阻塞，只有next方法返回了null时，才会跳出循环，消息循环才会中止。如果有消息的话，会调用Handler（msg的target）的dispatchMessage()方法接收消息，这时消息被交给了目标Handler所在的线程中处理，即完成了跨线程。

7. Handler

​ Handler中发送消息有两种方式post和send，post将Runnable对象发送出去，send直接发送Message。

那么接收消息便一定要判断类型再去执行。

  1. 发送消息

    post：

    post方式的方法中调用了3个方法sendMessageAtTime()、sendMessageDelayed() 、sendMessageAtFrontOfQueue()

    sendMessageAtTime()：在确定的时间发送消息。

    sendMessageDelayed()：延迟入队，其最后调用了sendMessageAtTime()，并在uptimeMillis参数处传入了SystemClock.uptimeMillis() + delayMillis。SystemClock.uptimeMillis() 是获取系统从开机到现在的时间，期间不包括休眠，这是一个相对时间。

    sendMessageAtFrontOfQueue()：将消息加入消息队列队首，通过将uptimeMillis参数置0调用enqueueMessage()实现。

**send：**

send方式也是调用了三种，与post类似。这些方法有sendMessage与sendEmptyMessage的区分，sendEmptyMessage顾名思义就是新建一个Message对象，将传入的what赋给新的Message对象的what，再将其入队。

  1. 接收处理消息

    dispatchMessage()方法首先会判断三种情况。

    1. post发送来的，那么会执行Message中的Runnable的run方法。
    2. 创建Handler时传入了一个Callback，那么会调用这个Callback的handleMessage()方法。
    3. 直接调用Handler的handleMessage()，这是一个空方法，具体由重写实现。

    在执行完任务后，跨线程通讯完成。
