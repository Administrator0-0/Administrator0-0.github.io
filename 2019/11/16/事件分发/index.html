安卓事件分发机制

安卓事件分发机制

1.整体

- U型，Activity → ViewGroup → View→ ViewGroup → Activity

2.过程

  • 首先由Activity 的dispatchTouchEvent()分发。

DecorView的superDispatchTouchEvent()方法中调用了父类的dispatchTouchEvent()，即调用了ViewGroup的dispatchTouchEvent()，事件传递到了ViewGroup。

  • 当ViewGroup的dispatchTouchEvent()调用时，先判断是否拦截，不拦截则分发给子View，拦截则通过父View的dispatchTouchEvent()，自己处理事件。

    View的dispatchTouchEvent()中，若若OnTouchListener不为空，当前View为enable，mOnTouchListener.onTouch(this, event)回调时返回true（即回调view注册监听中的onTouch()返回true），则返回true，事件停止传递，否则由View处理。

    View通过调用onTouchEvent()自身处理事件

    当View的返回true时，ViewGroup和Activity都返回true，事件停止传递，若View返回false，ViewGroup调用父类的dispatchTouchEvent()，若返回true则Activity也true，事件停止传递，若返回false，Acitivity调用onTouchEvent()自身处理，若点击事件在边界外，则返回true，事件消费，否则返回false，事件未消费。
